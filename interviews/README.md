# Interview Retrospectives

This folder contains my interview retrospectives. After each interview, I document:

- topics I couldn't explain confidently;
- knowledge gaps that became apparent;
- questions that surprised me;
- concepts I want to understand from first principles;
- a concrete learning plan.

The goal is not to record interview outcomes, but to build a structured roadmap for continuous improvement.

Each retrospective is written as an engineering postmortem: identify gaps, define action items, learn, and revisit.

## Structure

Each retrospective typically includes:

- Overall impression
- Knowledge gaps identified
- Things I thought I understood but didn't
- Questions that surprised me
- Learning plan
- Biggest lessons
- Action items

## Philosophy

I believe technical interviews are one of the fastest ways to discover blind spots.

Instead of treating them as pass/fail events, I use them as opportunities to improve my understanding of machine learning, LLMs, software engineering, and AI systems.
